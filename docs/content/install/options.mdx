# Options & Environment Variables

Set any env var to skip interactive prompts and run non-interactively.

## Variables

| Variable | Options | Default |
|----------|---------|---------|
| `SCOPE` | `project`, `global` | `project` |
| `DEPT` | `engineering`, `all`, `sales`, `marketing`, `hr`, `l-and-d`, `finance`, `operations`, `admin`, `leadership`, `product-design`, `project-management` | `engineering` |
| `TOOLS` | `auto`, `all`, `claude`, `cursor`, `codex`, `amp`, `windsurf`, `gemini` | `auto` |
| `REPO_URL` | Any git URL | `https://github.com/stratpoint-engineering/ultimate-ai-agents.git` |
| `BRANCH` | Any git branch | `main` |

## Examples

```bash
# Install only engineering agents for Claude Code
TOOLS=claude DEPT=engineering ./install.sh

# Install everything globally
SCOPE=global DEPT=all TOOLS=all ./install.sh

# Install sales agents for Claude + Cursor
TOOLS=claude,cursor DEPT=sales ./install.sh

# Install from a fork
REPO_URL=https://github.com/myorg/ultimate-ai-agents.git ./install.sh

# Install from a feature branch
BRANCH=feature/new-agents ./install.sh
```

## Scope: Project vs Global

| | Project | Global |
|-|---------|--------|
| **Location** | `./.claude/`, `./.cursor/` | `~/.claude/`, `~/.cursor/` |
| **Visibility** | Current directory only | All projects |
| **Use case** | Per-project agent sets | Shared across everything |

## Tool Auto-Detection

When `TOOLS=auto` (default), the installer checks for each tool:

| Tool | How it's detected |
|------|-------------------|
| Claude Code | `claude` command or `~/.claude/` exists |
| Cursor | `cursor` command or `~/.cursor/` exists |
| Codex CLI | `codex` command or `~/.codex/` exists |
| Amp Code | `amp` command or `~/.amp/` exists |
| Windsurf | `~/.codeium/` exists |
| Gemini CLI | `gemini` command or `~/.gemini/` exists |
